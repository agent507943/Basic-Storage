Lab Steps

Step 1: Create ACL to permit DNS and HTTP

Create standard/extended ACL permitting tcp 80 and udp/tcp 53 from user subnet to server and deny others.

Expected Commands:
  • ip access-list extended USER_TO_SRV
    permit tcp 10.1.1.0 0.0.0.255 host 10.2.2.10 eq 80
    permit udp 10.1.1.0 0.0.0.255 host 10.2.2.10 eq 53
    permit tcp 10.1.1.0 0.0.0.255 host 10.2.2.10 eq 53
    deny ip any any

Step 2: Apply ACL to interface

Apply the ACL in the correct direction on the router interface facing users.

Expected Commands:
  • interface gigabitEthernet0/1
    ip access-group USER_TO_SRV in

Step 3: Verify access

From a user host attempt: ping server, curl http://10.2.2.10, and dig @10.2.2.10 www.example.com

Expected Commands:
  • ping 10.2.2.10
  • curl http://10.2.2.10
  • nslookup www.example.com 10.2.2.10

Simulated Outputs

  • curl http://10.2.2.10 ‚Üí http:200 OK
  • nslookup www.example.com 10.2.2.10 ‚Üí answer:10.2.2.10
